## What goenv does

Many tools consume `.env` files, but working with them from a shell script or CI pipeline is awkward. goenv fills that gap:

- **Check existence** — exit with a non-zero code if a required variable is missing, making it useful as a guard in scripts.
- **Write variables** — add or remove variables from a file, creating the file if necessary.
- **Convert formats** — transform a `.env` file into JSON, YAML, TOML, XML, or INI for tools that don't speak `.env`.
- **Read typed values in Go** — call `env.String`, `env.Int`, `env.Bool`, or `env.Duration` instead of calling `os.Getenv` and parsing the result yourself.

## Supported .env file conventions

goenv recognises the standard set of environment file names used by Node.js, Docker Compose, and similar tooling:

| File | Purpose |
|------|---------|
| `.env` | Default environment file |
| `.env.local` | Local developer overrides, not committed to source control |
| `.env.development` | Development environment settings |
| `.env.production` | Production environment settings |

<Note>
Files whose path contains `prod` are treated as production files and are write-protected by default. Pass `-prod` explicitly, or set `GOENV_NEVER_WRITE_PRODUCTION=false` in your environment, to override this behaviour.
</Note>

## Installation

### CLI

Install the `goenv` binary with `go install`:

```bash
go install github.com/andreimerlescu/goenv@latest
```

The binary is placed in `$GOPATH/bin` (or `$HOME/go/bin`). Make sure that directory is on your `$PATH`.

### Go library

Add the `env` package to your module with `go get`:

```bash
go get github.com/andreimerlescu/goenv/env
```

Then import it in your Go code:

```go
import "github.com/andreimerlescu/goenv/env"
```
