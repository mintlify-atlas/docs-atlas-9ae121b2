`goenv` gives you a fast, scriptable way to work with `.env` files from the command line — and a clean Go package for type-safe environment variable access in your applications.

Whether you need to check if a key exists, add new variables, or export your `.env` to JSON, YAML, TOML, XML, or INI, `goenv` handles it with simple flag-based commands and exit codes designed for shell scripting.

## Installation

<Steps>
  <Step title="Install the CLI">
    ```bash
    go install github.com/andreimerlescu/goenv@latest
    ```
  </Step>
  <Step title="Create or initialize an env file">
    ```bash
    goenv -init -file app.env
    ```
  </Step>
  <Step title="Add your first variable">
    ```bash
    goenv -file app.env -add -env APP_ENV -value production -write
    ```
  </Step>
  <Step title="Verify it was written">
    ```bash
    goenv -file app.env -print
    # APP_ENV=production
    ```
  </Step>
</Steps>

## Use the Go library

Import the `env` package directly in your Go applications for type-safe access to environment variables:

```go
import "github.com/andreimerlescu/goenv/env"

host := env.String("HOSTNAME", "localhost")
port := env.Int("PORT", 8080)
debug := env.Bool("DEBUG", false)
timeout := env.Duration("TIMEOUT", 5*time.Second)
```

<Note>
  The `env` package reads from the runtime environment (not from `.env` files directly). Use the `goenv` CLI to load `.env` files into your shell before running your application.
</Note>
